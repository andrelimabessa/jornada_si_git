O que é o GIT ?

O GIT é um Sistema de Controle de Versão

O que é sistemas de vontrole de versão ?

"Um sistema de controle de versão ter por função registrar alterações em um arquivo ou conjunto de arquivos ao longo do tempo para que você possa lembrar versões específicas mais tarde."

Sistema de Controle de Versão (VCS)

Sistema de Controle de Versão Centralizado (CVCS)

Sistema de Controle de Versão Distribuído (DVCS)

Caso 1: Versionar Projeto

  • Eu como usuário quero por um projeto sob controle de versão
  • Entro em um diretório
  • Configuro controle de versão

Controle de versão centralizado

Cenário 1: Criando Repositório Local


							mkdir nova-pasta    /* cria uma nova pasta */
						

							cd nova-pasta       /* entra em uma nova pasta */
						

							git init

Caso 2: Commit Arquivo

  • Eu como usuário quero commitar um arquivo
  • Entro em um diretório sob controle de versão
  • Crio um arquivo
  • Edito o arquivo
  • Commito o arquivo

Pré-Requisito: Ter um arquivo para commitar


							pwd     /*indica diretório corrente*/

Área de Trabalho e Área de Preparação

Ciclo de Vida de um Aquivo

Cenário 1: Qual Estado dos Arquivos 


							git status

Cenário 2: Adicionar Arquivos na Área de Preparação


							git add arquivo
						

							git add -A
						

							git status

Dica: .gitignore

  • Crie mais um arquivo com um nome qualquer
  • Cria um arquivo chamado .gitignore
  • Adiciona nele o nome do arquivo criado no primeiro passo
  • Teste de novo o git status

Cenário 3: Commitando Arquivo


							git commit -m "Comentário"
						

							git status

Dica: rm 


							git rm nome-arquivo

Dica: Commite sem Staging Area (Cuidado) 


							git commit -a -m "Comentário"

Caso 3: Desfazer Alterações

  • Eu como usuário quero desfazar alguma alteração
  • Entro em um diretório sob controle de versão
  • Indentifico um artefato que desejo que volte a uma condição anterior

Pré-Requisito: Altere dados em um arquivo que já se encontra em controle de versão

Dica: diff

Para ver as alterações em todos os arquivos 

							git diff

Cenário 1: Descartar Alteração no Diretório de Trabalho


							git checkout HEAD arquivo

Pré-Requisito: Altere dados em um arquivo que já se encontra em controle de versão 


							git add arquivo

Dica: diff

Para ver as alterações na Staging Area 

							git diff --staged

Cenário 2: Descartar Alteração na Área de Preparação


							git reset HEAD arquivo

Caso 4: Gerenciar Branches - Básico

  • Eu como usuário desejo gerenciar branches
  • Entro em um diretório sob controle de versão
  • Identifico os branches existentes
  • Crio um branche novo
  • Entro no novo branche
  • Commito as alterações para o branch criado

Cenário 1: Identificar Branches Existentes


							git branch

Cenário 2: Criar um Novo Branch


							git branch nome_do_branch

Cenário 3: Entrar no Novo Branch


							git checkout nome_do_branch

Caso 5: Gerenciar Branches - Merge Simples

  • Eu como usuário desejo mesclar dois branches
  • Crio e commito um arquivo no primeiro branch
  • Crio um arquivo no segundo branch (o commit é opcional nesse momento)
  • Mesclo o primeiro branch com o segundo

Pré-Requisito 1: Ter dois branches com arquivo em comum

Pré-Requisito 2: Ter o arquivo comum alterado em um dos branches

Cenário 1: Mesclando dois branches

Entro no segundo branch 

							git merge nome_branch_de_onde_vem_itens_para_mesclar_com_o_corrente

Caso 6: Gerenciar Branches - Merge com Conflito

  • Eu como usuário desejo mesclar dois branches com um arquivo em comum
  • Edito o arquivo comum e commito no primeiro branch
  • Entro no segundo branch
  • Edito o arquivo comum, sem commitar, no segundo branch
  • Tento mesclo o segundo branch com o primeiro

Pré-Requisito 1: Criar branch

Pré-Requisito 2: Alterar arquivo comum no branch original

Pré-Requisito 3: Commitar arquivo comum

Pré-Requisito 4: Entrar no segundo branch

Pré-Requisito 5: Alterar arquivo comum sem commitar

Cenário 1: Mesclando Dois Branches


							git merge nome_branch_de_onde_vem_itens_para_mesclar_com_o_corrente

Ver Mensagem

Saída 1: Commita o arquivo no branch corrente e depois faz o merge

Saída 2: Uso de Stash


							git stash
						

							git merge nome_branch_de_onde_vem_itens_para_mesclar_com_o_corrente
						

							git stash pop

Caso 7: Time de Desenvolvimento

  • Eu como usuário desejo acessar o repositório compartilhado do meu time de desenvolvimento
  • Devo clonar o repositório
  • Devo manter meu respitório local sincronizado

Cenário 1: Clonar Repostório 


							git clone https://github.com/andrelimabessa/jornada_si_git.git

Pré-Requisito 1: Criar branch com seu nome

Pré-Requisito 2: Entrar no seu branch

Pré-Requisito 3: Realize alguma alteração nele

Pré-Requisito 4: Commite a alteração localmente

Cenário 2: Enviar Alterações para o Repositório Remoto


							git push origin nome_branch_corrente

Cenário 3: Buscar Atualizações do Repositório Remoto


							git pull
